Your token is never exposed.
Here's the technical proof.

We built this with maximum paranoia. The encryption isn't cosmetic.

🔐
Encrypted immediately on save

When you click "Save & Secure Token", your token is sent over HTTPS to our server, where it is immediately encrypted with AES-256-GCM and a random 12-byte IV. The plaintext is destroyed in memory.

🗄️
Only ciphertext in the database

We store three things: the encrypted ciphertext, a random IV, and a GCM auth tag. Without the ENCRYPTION_KEY (which is only in our server environment), this data is useless to anyone.

⚡
Decrypted only in volatile memory

During a vote execution, your token is decrypted inside a Node.js process on Vercel or GitHub Actions, used for a single HTTP call, then garbage-collected. It never touches disk or logs.

👤
Admin cannot read your token

Even the service owner cannot decrypt your token without direct server access. Admin actions (approve/disable) never touch the encrypted token columns in the database.
